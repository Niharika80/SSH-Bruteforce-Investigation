# INCIDENT RESPONSE REPORT
Incident ID : SOC-LAB-1  
Analyst: Niharika  
Date: 24/02/2026  

## EXECUTIVE SUMMARY:

On 24 February 2026, a suspicious authentication activity was detected on a monitored Linux endpoint through Wazuh. The investigation revealed that an attacker from IP address 10.0.2.30 performed a brute-force attack against the SSH service of the target machine 10.0.2.20. After multiple failed login attempts, the attacker successfully authenticated to the system using valid credentials for the user phoebe. Following the successful login, the attacker executed privilege escalation commands and obtained root-level access on the compromised system. The attack activity was identified by analyzing authentication logs (auth.log) and correlated alerts generated by Wazuh. Based on the investigation, the incident has been classified as High severity due to unauthorized access and privilege escalation. Immediate mitigation actions such as enforcing strong passwords, restricting SSH authentication methods, and reviewing sudo privileges are recommended to prevent similar incidents in the future.

Severity: High

## TECHNICAL ANALYSIS

1.How Bruteforce works?  
The attacker performed a brute-force attack by attempting multiple password combinations using an automated tool against the SSH authentication service. This resulted in several failed login attempts before a valid credential was discovered.

2.Why Successful Login happened?  
The attacker successfully authenticated after identifying the correct credentials for the user phoebe, allowing remote access to the system via SSH.

3.Why sudo escalation was possible?  
The attacker was able to escalate privileges because the compromised user had sudo permissions configured on the system. By executing privileged commands, the attacker obtained root-level access.

## INDICATORS OF COMPROMISE

Attacker IP Address: 10.0.2.30  
Compromised User Account: phoebe  
Log Source: /var/log/auth.log  
Attack Vector: SSH Brute Force  

## ACTIONS
Escalate to L2 Analyst
Marked the alert as True Positive

## RECOMMENDATIONS

1. Disable password-based SSH authentication and enforce SSH key-based authentication.
2. Implement strong password policies to prevent brute-force attacks.
3. Restrict unnecessary sudo privileges and follow the principle of least privilege.
4. Enable account lockout after multiple failed login attempts.
5. Monitor authentication logs regularly for suspicious activity.
